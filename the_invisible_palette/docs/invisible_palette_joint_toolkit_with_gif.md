# Invisible Palette Joint Toolkit with GIFs

This document covers:

```bash
python src/invisible_palette_joint_toolkit_with_gif.py
```

This script is the cleaned-up joint comparison track.

It preserves the original spirit:

- the same urn-generation choices,
- the same prior over `C`,
- one fixed `--alpha` for the fixed Dirichlet model,
- one comparison run over three inference views,
- GIFs generated by default.

The three compared views are:

1. `full_uniform`
2. `full_dirichlet`
3. `joint_c_alpha`

---

## 1. Philosophy

This script is intentionally narrower than the earlier joint branch.

It does not try to be:

- a policy benchmark,
- a budget sweep tool,
- a parameter playground.

It does one thing:

> compare how three increasingly flexible assumptions learn the hidden support size from the same sample stream.

That is the right scope for the main comparison script.

---

## 2. The Three Compared Models

### `full_uniform`

Uses the full occupancy counts and assumes equal colour probabilities under candidate support size `C`.

### `full_dirichlet`

Uses the full occupancy counts and a fixed symmetric Dirichlet-multinomial model with:

```bash
--alpha
```

This matches the original fixed-alpha Dirichlet comparison.

### `joint_c_alpha`

Uses the same Dirichlet-multinomial family, but learns over a built-in grid of candidate alpha values and infers:

\[
p(C,\alpha \mid \mathbf{x})
\]

The alpha grid is intentionally internal so the CLI stays clean.

---

## 3. Inputs

### Explicit counts

You can still specify the hidden urn directly:

```bash
--counts 4,3,7
```

### Generated urns

Or generate the urn using the same choices as the original scripts:

- `--m`
- `--count-mode`
- `--min-count`
- `--max-count`

So the script still supports:

- explicit count vectors,
- generated uniform count profiles,
- generated skewed count profiles,
- generated one-heavy count profiles.

---

## 4. Prior on `C`

This is kept the same as the original toolkit:

- `--prior-type uniform`
- `--prior-type geometric`

with:

- `--prior-lam`

used only when the prior is geometric.

Important:

- this prior is the prior on hidden support size `C`
- `Dirichlet` is not a `prior-type` choice for `C`
- the Dirichlet part belongs to the observation model, controlled here by fixed `--alpha` in the fixed model and by the internal alpha grid in the joint model

## 7. Recommended Runs

### Explicit counts

```bash
python src/invisible_palette_joint_toolkit_with_gif.py \
  --counts 4,3,7 \
  --batch-size 6 \
  --rounds 20 \
  --seed 0 \
  --c-max 12 \
  --alpha 0.5 \
  --prior-type uniform \
  --prior-lam 0.2 \
  --gif-fps 2.0 \
  --outdir results/joint_ex_437
```

### Generated counts

```bash
python src/invisible_palette_joint_toolkit_with_gif.py \
  --m 8 \
  --count-mode uniform \
  --min-count 1 \
  --max-count 10 \
  --batch-size 8 \
  --rounds 20 \
  --seed 0 \
  --c-max 20 \
  --alpha 0.5 \
  --prior-type uniform \
  --prior-lam 0.2 \
  --gif-fps 2.0 \
  --outdir results/joint_results
```

```bash
python src/invisible_palette_joint_toolkit_with_gif.py \
  --m 8 \
  --count-mode uniform \

  --max-count 10 \
```

- all CLI arguments used
